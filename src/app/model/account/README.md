# Account Model

このパッケージは、アカウントに関するドメインルールを扱う。
永続化、API 入出力、認証処理そのものは責務に含めない。

## 設計方針

- `Account` は権限を割り当てられる主体を表す。
- プロフィール、基本設定、外部認証 ID、メタデータは `principal_id` で紐づく別エンティティとして扱う。
- アカウント削除は物理削除ではなく、`DELETED` への状態変更と個人情報のマスクで表現する。
- `DELETED` になったアカウントは `ACTIVE` / `SUSPENDED` に戻せない。
- プロフィール・基本設定の更新では、実際に値が変わった場合のみ `metadata.updated_at` を更新する。

## 注意点

- メールアドレス検証は現状 `@` の有無のみ。
- 認証プロバイダは現状 Firebase のみ。
- トランザクション管理はこの層では扱わない。
